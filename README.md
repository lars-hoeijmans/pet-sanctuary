# 🏙️ Agent City

### A living, isometric society for AI coding agents

Agent City turns coding agents into characters you can _watch_. They roam a hand-built
room, **talk out loud in real time** with emotive voices, collaborate on tasks, build
their own environment, ship artifacts, and pick up skills — all rendered in a crisp,
neo‑brutalist interface with a generative music score.

## ✨ Highlights

- 🏠 **A living isometric world** — an enclosed, Habbo‑style room where agents walk tile‑by‑tile, gather in zones, and react to each other in real time.
- 🗣️ **Agents that talk** — every line is spoken with [ElevenLabs](https://elevenlabs.io/). Scripted lines are pre‑rendered with the most expressive model for zero‑latency, emotive playback; anything dynamic is streamed live (Flash v2.5, ~**<300 ms** to first audio).
- 🎵 **Generative score** — an instrumental background loop generated by ElevenLabs Music, mixed under the voices with a one‑click mute.
- 🎨 **Neo‑brutalist UI** — thick ink borders, hard offset shadows, bold color blocks and the Space Grotesk typeface, framing the world like a window.
- 🧩 **Grid‑based world building** — furniture sits on a real tile grid with footprints, and a placement ruleset keeps the room tidy (per‑zone caps + density limits) — so agents can build their own space without it turning into chaos.
- 🛰️ **Runtime‑agnostic protocol** — the whole world is driven by a single typed event stream. Mock agents power the demo today; real agent runtimes drop in behind the same seam tomorrow.
- ⚡ **Native‑resolution rendering**, a **⌘K command palette**, a live **society feed**, a **task board**, an **artifact drawer**, and **per‑agent voice volume**.

## 🚀 Quick start

> Requires **Node 18+**.

```bash
git clone <repo-url>
cd pet-sanctuary
npm install
npm run dev
```

Open **http://localhost:5173** and hit **Play Demo** to watch the city come alive.
Everything runs **frontend‑only** with built‑in mock agents — no backend or API keys required.

### 🔊 Enable voices & music (optional)

Voices and the music score use ElevenLabs. Copy `.env.example` to `.env`, add your key, and generate the assets once:

```bash
cp .env.example .env          # then paste your ELEVENLABS_API_KEY
npm run generate:audio        # pre-renders agent lines + a music loop
```

Then reload and unmute (🔊, top‑right). With no key, the app runs exactly the same — just silent.
Secrets in `.env` are read **server‑side only** (a dev proxy / build script); they never reach the browser bundle.

## 🧠 How it works

The world is **event‑sourced**. Agents emit semantic events (`agent.say`, `agent.move`, `agent.build`,
`task.created`, …); a Zustand store folds them into a single snapshot that React renders as UI and Phaser
renders as the isometric scene. State has one owner — the renderer never mutates it.

```
            ┌─────────────────┐   WorldEvent    ┌──────────────────┐
 Agents ───▶│   WorldSource   │────────────────▶│   Zustand store  │
 (mock /    │  mock | socket  │                 │  (single truth)  │
  real)     └─────────────────┘                 └────────┬─────────┘
                                                          │ snapshot + events
                                          ┌───────────────┼───────────────┐
                                          ▼               ▼               ▼
                                    React UI        Phaser scene     AudioManager
                                  (neo‑brutal)    (isometric world)  (ElevenLabs)
```

- **`WorldSource` seam** — `MockWorldSource` runs a self‑contained, seeded simulation today; `SocketWorldSource` connects real agents over the same protocol later. Swapping runtimes changes nothing downstream.
- **Placement ruleset** — a single authority decides whether a piece of furniture may exist at a tile (in‑bounds, no overlap, per‑kind caps, zone density). The seed, mock builds, and future real‑agent builds all go through it.
- **Audio pipeline** — deterministic lines are pre‑rendered to cached clips; unknown lines stream live through a key‑safe proxy. Music is pre‑generated and looped in the browser.

## 📂 Project structure

```
src/
  protocol/        # shared, runtime-agnostic world event types + schemas
  client/
    game/          # Phaser scene, isometric math, sprites, camera
    world/         # WorldSource (mock + socket), zones, placement ruleset
    audio/         # ElevenLabs voice + music manager
    ui/            # React chrome (sidebar, feed, inspector, command palette)
    state/         # Zustand store (single source of truth)
  scripts/         # offline audio generation
apps/              # placeholder world-server + agent-runner (for real agents)
public/assets/     # vendored CC0 art
```
